Algo Trading Platform

A collaboration opportunity for QF graduate students

This is an outline for a project where we build an algo trading platform. The goal is hands-on experience with financial data and technology: data engineering, data analysis, algo trading, and compilers.

"Quant finance" programs contrast with "computational finance" programs that involve more interfacing with markets using code—understanding the technical infrastructure of exchanges. Baruch's MFE program is rated #1. partly because they emphasize programming; graduates disproportionately work as quant devs, which has higher minimum salaries. Even if you don't aspire to be a quant developer, understanding market microstructure matters for taking liquidity and orderflow into account.

Precedent

Quantopian was a platform where users developed and shared algo trading strategies in Python. You could backtest against historical data and simulate P&L with walk-forward tests. Point72 (Steve Cohen) was an investor; the business model was crowd-sourced hedge fund—pick the best strategies, trade with their capital, share profits with authors. They shut down in 2020.

"To fill the gap left by Quantopian, traders have relied on open source projects like Gekko and Freqtrade. The first companies to capitalize were CrunchDAO, QuantConnect, and Numerai. Recently, cloud-based tools emerged for crypto: 3commas, Mudrex, and Tuned."
— Wikipedia

The space remains niche and the tech last-gen. Meanwhile, there's a confluence of factors:

  • Millions of laid-off tech workers with time, money, and skills to trade
  • Oversupply of CS grads; employers favor senior hires who can harness AI
  • QF going mainstream—no longer esoterica from The Big Short
  • Retail traders worldwide dialing in (Korean senior citizens taking stock trading courses, etc.)
  • Proliferation of new markets: Polymarket, crypto, prediction markets
  • A US administration that doesn't shy away from market-moving headlines

Interest in quant trading is higher than ever. Competition for institutional jobs is fierce. Groups aspiring to start hedge funds from dorm rooms garner millions of views. AI has enabled non-software engineers to build non-trivial applications. This project: let non-engineers build their own algo trading strategies. Robinhood democratized trading for retail; this is an attempt at democratizing algo trading.

The Simulator

In Dune, characters possess prescience—the ability to see the future. Retail traders have a similar fantasy: travel back in time, knowing what you know now, and make trades. People joke about this on WallStreetBets. What if it doesn't have to be a joke?

I built a prototype "game" where you're back in January 2020 before the GameStop short squeeze with $1,000. Each trading day, you decide: buy, sell, or hold.

A little 200% gain here, a 500% gain there, and a few hundred clicks later you've turned $1,000 into millions. What if instead of daily, this was hourly? Every minute? Higher resolution data opens new possibilities.

Realism

The simulator assumes perfect liquidity. Real trading involves:

  • Slippage: You want to sell 100,000 shares at $X. Only 50,000 find buyers.
  • Market impact: Selling 1,000,000 shares? Participants notice and back away.
  • Execution: Algorithms must handle partial fills and recalibrate positions.

There's meaningful work to make this simulator realistic—potentially overlapping with simulation coursework.

From Clicking to Code

Instead of manually clicking, specify code that clicks for you. Implement a function that takes today's prices and returns a decision. We call your function on each day of the simulation.

Voil√†: a crude algo trading backtesting platform. It doesn't have to be limited to simulation. Link a brokerage account, execute the user-specified function once per trading day—now we have a crude live trading platform.

Why JavaScript

We're using JavaScript instead of Python for the algo trading language. JavaScript is the most popular programming language by many metrics, yet there's no finance ecosystem around it. There are millions of webdevs who'd try algo trading if the barrier were lower.

The Python ecosystem

Python has NumPy, Pandas, and now Polars (Optiver is a sponsor). At Jane Street, one intern project was porting Polars to OCaml. If we port Polars to JavaScript, that's a talking point for interviews at firms using Polars.

Developer mindshare

Python is popular for data work because people created data libraries for it, then a community formed. This is developer mindshare—an anthropological phenomenon, not Python being fundamentally superior. Stats/ML/finance libraries don't exist for JavaScript yet. We have the opportunity to pioneer them.

If developer mindshare shifts toward JavaScript for finance, you're a key figure in the supply chain. Jane Street famously hired most of the OCaml ecosystem contributors.

Compilers

Another workstream: taking users' JavaScript algos and converting them to something fast—C++ (lingua franca of quant finance) or Rust (gaining popularity). Working with programming languages at this meta level is compilers work.

Most of Jane Street's technical interns work on language/compilers projects.

C++

Every HFT fund uses C++. Stock exchanges are written in C++. Various aspects of this project will involve C++ code:

  • High-performance backtesting engine
  • Exchange connectivity and market data parsing
  • Compilation targets for user strategies
  • Latency-critical execution paths
